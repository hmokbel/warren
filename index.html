Plaid API Usage

How Warren Uses Plaid

Warren uses the Plaid API to facilitate secure bank account connections. Plaid is an independent third-party service; Warren is not affiliated with, endorsed by, or officially partnered with Plaid Technologies, Inc.

  1. 1

    User Initiates Connection

    The user taps "Connect Account" within Warren. No financial data is accessed until the user explicitly consents.

  2. 2

    Plaid Link Opens

    Warren launches Plaid Link, a secure, Plaid-hosted UI. Warren never sees, handles, or stores the user's bank username or password at any point.

  3. 3

    Authentication at the Bank

    The user authenticates directly with their financial institution inside Plaid Link. Multi-factor authentication (MFA), if required by the bank, is handled entirely within Plaid Link.

  4. 4

    Plaid Returns a Token

    Upon successful authentication, Plaid returns a short-lived public_token to Warren's mobile client. This token has no value on its own and cannot be used to access account data directly.

  5. 5

    Secure Server-Side Exchange

    Warren's mobile app sends the public_token to Warren's backend API over HTTPS/TLS. The backend exchanges it for an access_token using the Plaid API. The access_token is stored securely server-side and is never exposed to the client or user.

  6. 6

    Data Retrieval and Analysis

    Warren's backend retrieves transaction, balance, and account data from Plaid on the user's behalf. This data is then processed to generate spending summaries and financial insights displayed within the app.

Security

Data Security

Warren is built with security as a foundational requirement, not an afterthought. Below are the specific measures in place to protect user data.

  • ✓

    No Credential Storage

    Warren never collects, stores, or transmits your bank username or password. Authentication is handled entirely by Plaid Link and your financial institution.

  • ✓

    Encrypted Transport

    All data transmitted between the Warren app, Warren's backend, and the Plaid API is encrypted using HTTPS/TLS.

  • ✓

    Server-Side Secrets

    Plaid API credentials (client ID and secret) are stored exclusively on Warren's backend server. They are never embedded in the mobile app or exposed to end users.

  • ✓

    Explicit User Consent

    No financial data is accessed without a clear, explicit user action. Users must actively choose to connect an account.

  • ✓

    Data Minimization

    Warren requests only the Plaid API products and account data necessary to provide the app's core features. No unnecessary data is collected.

  • ✓

    Secure Backend Architecture

    All Plaid API calls that require credentials are made server-side. The mobile app communicates with Warren's backend, which acts as a secure intermediary.

Privacy Policy

Effective date: May 12, 2026  ·  Owner: Husein Mokbel

1. Information We Collect

When you connect a bank or investment account, Warren — via Plaid — retrieves account and transaction data (account names, balances, transaction descriptions, amounts, and dates) that you have authorized us to access. We do not collect your bank login credentials at any time.

We may also collect basic account information you provide when registering (such as your name and email address) and standard usage logs for debugging and performance monitoring.

2. How We Use Your Information

Your financial data is used exclusively to provide Warren's core functionality: categorizing transactions, calculating balances, and generating personalized financial insights within the app. We do not sell your data, share it with advertisers, or use it for any purpose unrelated to delivering the service.

3. Plaid's Role

Warren uses Plaid Technologies, Inc. to facilitate bank account connections. By connecting an account, you also agree to Plaid's End User Privacy Policy. Plaid's privacy practices are independent of Warren's.

4. Data Sharing

We do not sell, rent, or trade your personal or financial information to third parties. Data may be shared only with service providers necessary to operate Warren (such as cloud infrastructure and the Plaid API), and only to the extent required to provide the service. Any such third-party providers are subject to their own terms of service and privacy policies.

5. Cookies and Tracking

The Warren website (this page) does not use cookies, tracking pixels, or any third-party analytics scripts. No personal data is collected through this website.

6. Security Measures

We implement industry-standard security measures including HTTPS/TLS encryption in transit and secure, access-controlled storage for sensitive data. Plaid API secrets are stored server-side only and are never exposed to client applications or end users.

7. Children's Privacy

Warren is not directed to children under the age of 13. We do not knowingly collect personal information from children under 13. If you believe a child has provided us with personal information, please contact us so we can delete it.

8. Changes to This Policy

We may update this Privacy Policy from time to time. Material changes will be communicated via the app or by updating the effective date above. Continued use of Warren after changes take effect constitutes acceptance of the updated policy.

Data Retention & Disposal Policy

Data Retention and Disposal Policy

Effective Date: March 15, 2026  ·  Organization: Husein Mokbel (Founder / Developer)

1. Purpose

This Data Retention and Disposal Policy defines how consumer data is collected, stored, retained, and securely deleted in compliance with applicable data privacy laws and industry best practices. The goal is to ensure that data is retained only as long as necessary and disposed of securely when no longer required.

2. Scope

This policy applies to all consumer data processed by Warren, including data retrieved via the Plaid API, as well as any associated metadata stored in backend systems.

3. Data Collection and Use

  • Consumer financial data is accessed exclusively through the Plaid API following explicit user consent.
  • Data is used solely for providing core application functionality.
  • Warren adheres to the principle of data minimization, collecting only what is necessary.

4. Data Retention

  • Consumer data is retained only for as long as required to fulfill the purpose for which it was collected.
  • Data retention periods are reviewed periodically and adjusted based on operational and legal requirements.
  • Inactive or unused data is identified and subject to scheduled deletion.

5. User-Initiated Deletion

  • Users may request deletion of their data at any time by contacting mokbel.huss@gmail.com.
  • Upon request, all associated consumer data is permanently removed from active systems within a reasonable timeframe.
  • Account deletion triggers full removal of linked data.
  • You may also disconnect any linked account at any time from within the Warren app. Upon disconnection, the associated Plaid access token is removed, preventing further data retrieval from that institution.
  • You may independently revoke Warren's access to your financial institution data through Plaid's consumer portal at my.plaid.com.

6. Automated Data Lifecycle Management

  • Systems are designed to periodically review stored data and remove data that is no longer necessary.
  • Temporary and cached data are automatically cleared at defined intervals.

7. Data Disposal and Destruction

  • Data is permanently deleted using secure deletion methods to prevent recovery.
  • Logical deletion processes ensure removal from production systems, backups, and logs where applicable.
  • Any residual data is overwritten or securely purged in accordance with industry standards.

8. Security Controls

  • Data in transit is encrypted using TLS 1.2 or higher.
  • Access to stored data is restricted based on least-privilege principles.
  • Administrative access is limited to authorized personnel only.

9. Compliance

  • This policy is designed to align with applicable data protection regulations, including but not limited to Canadian privacy laws and general data protection principles.
  • Consumer data is handled responsibly and securely at all times.

10. Policy Review and Updates

  • This policy is reviewed periodically to ensure continued compliance with evolving regulatory requirements and industry standards.
  • Updates are made as necessary to reflect changes in application architecture or data handling practices.
